# 旅行推銷員問題（TSP）與爬山演算法實作

本專案使用 Python 實作爬山演算法（Hill Climbing Algorithm），用以尋找旅行推銷員問題（Traveling Salesperson Problem, TSP）的局部最佳解。專案中採用了 2-opt 局部搜尋機制來定義鄰居狀態，並透過高度函數的轉換進行路徑長度的優化。

## 功能特點

* 初始解生成：依據城市編號順序（1 到 n）建立初始迴路。
* 高度函數轉換：將極小化問題（尋找最短距離）轉換為極大化問題，高度定義為總行走距離乘以負一。
* 2-opt 鄰居函數：遍歷所有不相鄰的邊進行斷開與重連（反轉子陣列），確保包含邊界狀況的完整鄰居空間。
* 最陡上升爬山法：每一步皆評估所有鄰居，並選擇能使高度提升最多的最佳鄰居進行移動，直到抵達局部最高峰。

## 演算法邏輯說明

### 1. 鄰居機制（2-opt）
當演算法選定路徑中的兩個切點時，會將該區間內的城市走訪順序進行反轉。此操作在圖論上等同於斷開兩條原本不相鄰的邊，並將其交叉重新連接，從而產生一個新的鄰居解。

### 2. 終止條件與限制
本演算法屬於貪婪搜尋，當前狀態周圍所有鄰居的高度均小於或等於當前高度時，演算法即會終止。此時輸出的結果為局部最佳解（Local Optimum），而非全域最佳解（Global Optimum）。
